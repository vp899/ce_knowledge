## 2. 架构目标与约束

### 2.1 质量属性需求
| 质量属性 | 需求描述 | 优先级 |
|----------|----------|--------|
| 性能 | 启动时间 < 3s，响应时间 < 100ms | 高 |
| 可靠性 | MTBF > 30000h，支持自动恢复 | 高 |
| 安全性 | 安全启动，数据加密 | 高 |
| 可扩展性 | 支持模块化扩展 | 中 |
| 可维护性 | 支持 OTA 升级 | 高 |
| 功耗 | 待机 < 1mA，工作 < 100mA | 高 |

### 2.2 约束条件
| 约束类型 | 描述 |
|----------|------|
| 硬件约束 | MCU: STM32F407, Flash: 1MB, RAM: 192KB |
| 成本约束 | BOM < ¥50 |
| 功耗约束 | 电池续航 > 30 天 |
| 时间约束 | 6 个月内量产 |
| 标准约束 | 通过 CE/FCC 认证 |

## 3. 系统架构

### 3.1 架构图
```
┌─────────────────────────────────────────────────┐
│                   应用层                          │
│   ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│   │ 传感器   │ │ 通信管理 │ │ 用户界面 │        │
│   └──────────┘ └──────────┘ └──────────┘        │
├─────────────────────────────────────────────────┤
│                   中间件层                        │
│   ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│   │ 状态机   │ │ 协议栈   │ │ 数据管理 │        │
│   └──────────┘ └──────────┘ └──────────┘        │
├─────────────────────────────────────────────────┤
│                   OS 抽象层                       │
│   ┌──────────────────────────────────────┐       │
│   │ [[rtos-drivers|FreeRTOS]] / CMSIS-RTOS               │       │
│   └──────────────────────────────────────┘       │
├─────────────────────────────────────────────────┤
│                   驱动层                          │
│   ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│   │ GPIO/I2C │ │ SPI/UART │ │ ADC/DMA  │        │
│   └──────────┘ └──────────┘ └──────────┘        │
├─────────────────────────────────────────────────┤
│                   硬件层                          │
│   STM32F407 + 传感器 + 通信模块 + 电源           │
└─────────────────────────────────────────────────┘
```

### 3.2 模块职责
| 模块 | 职责 | 接口 |
|------|------|------|
| 传感器模块 | 数据采集、校准 | I2C/SPI |
| 通信管理 | BLE/WiFi 连接、数据传输 | UART/SPI |
| 状态机 | 设备状态管理 | 内部 API |
| 协议栈 | 通信协议处理 | 回调函数 |
| 数据管理 | 数据存储、检索 | Flash API |

### 3.3 数据流
```
传感器 → 数据采集 → 数据处理 → 数据存储 → 数据上传
   │         │           │           │           │
   └──── I2C ──── 处理算法 ──── Flash ──── BLE ───┘
```

## 4. 接口定义

### 4.1 硬件接口
| 接口 | 引脚 | 速率 | 用途 |
|------|------|------|------|
| I2C1 | PB6(SCL)/PB7(SDA) | 400kHz | 传感器 |
| SPI1 | PA5(CLK)/PA6(MISO)/PA7(MOSI) | 10MHz | Flash |
| UART1 | PA9(TX)/PA10(RX) | 115200 | 调试 |
| UART2 | PA2(TX)/PA3(RX) | 9600 | 通信模块 |

### 4.2 软件接口
```c
// 传感器接口
int sensor_init(void);
int sensor_read(SensorData_t *data);
int sensor_calibrate(CalibData_t *calib);

// 通信接口
int comm_init(void);
int comm_send(const uint8_t *data, uint16_t len);
int comm_register_callback(CommCallback_t cb);

// 存储接口
int storage_init(void);
int storage_write(uint32_t addr, const uint8_t *data, uint16_t len);
int storage_read(uint32_t addr, uint8_t *data, uint16_t len);
```

## 5. 关键设计决策

| 决策点 | 方案 A | 方案 B | 选择 | 理由 |
|--------|--------|--------|------|------|
| RTOS | FreeRTOS | RT-Thread | FreeRTOS | 生态成熟、AWS 支持 |
| 通信 | BLE | WiFi | BLE | 低功耗、配对简单 |
| 存储 | 内部 Flash | 外部 SPI Flash | 外部 | 容量大、擦写寿命长 |
| 升级 | 整包升级 | 差分升级 | 整包 | 实现简单、可靠性高 |

---

## 6. 线程设计

| 线程 | 优先级 | 栈大小 | 周期 | 职责 |
|------|--------|--------|------|------|
| main | 最高 | 1KB | 事件 | 初始化、错误处理 |
| sensor | 高 | 512B | 100ms | 传感器数据采集 |
| comm | 中 | 1KB | 事件 | 通信协议处理 |
| storage | 低 | 512B | 事件 | 数据存储 |
| idle | 最低 | 256B | - | 空闲、喂狗 |

---

## 7. 安全设计

### 7.1 安全启动
- Bootloader 验证固件签名
- 固件版本防回滚
- 安全存储密钥

### 7.2 通信安全
- BLE 配对加密 (LE Secure Connections)
- 数据传输 AES-128 加密
- 防重放攻击 (序列号)

### 7.3 数据安全
- 敏感数据加密存储
- 安全擦除机制
- 访问控制

---

## 8. 功耗设计

| 模式 | 电流 | 唤醒源 | 保持状态 |
|------|------|--------|----------|
| 运行 | 50mA | - | 全部 |
| 空闲 | 10mA | 定时器/中断 | 全部 |
| 睡眠 | 100μA | RTC/外部中断 | RAM/RTC |
| 深度睡眠 | 10μA | 外部中断 | 备份寄存器 |
| 关机 | 1μA | 按键 | 无 |

---

## 9. 错误处理

| 错误类型 | 处理策略 | 恢复方式 |
|----------|----------|----------|
| 传感器故障 | 重试 3 次 | 切换备用传感器 |
| 通信断开 | 自动重连 | 指数退避重试 |
| 存储满 | 覆盖最旧数据 | 循环存储 |
| 看门狗超时 | 系统重启 | 自动恢复 |
| 硬件异常 | 安全状态 | 需人工干预 |
```

## 2. 设计评审检查清单

### 架构评审检查清单
```
□ 需求覆盖
  - 所有功能需求已分配到模块
  - 所有非功能需求有对应策略
  - 需求追踪矩阵完整

□ 模块设计
  - 模块职责单一明确
  - 模块间耦合度低
  - 接口定义清晰
  - 数据流合理

□ 质量属性
  - 性能瓶颈已识别
  - 可靠性策略已定义
  - 安全机制已设计
  - 可扩展性考虑充分

□ 约束满足
  - 资源使用在限制内
  - 功耗预算合理
  - 成本控制可行
  - 进度计划合理

□ 风险识别
  - 技术风险已识别
  - 缓解措施已定义
  - 第三方依赖已评估
```
