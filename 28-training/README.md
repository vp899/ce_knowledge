## Level 0: 入门基础 (4 周)

### 课程 0.1: 电子基础
```
Week 1: 模拟电路
├── 电阻/电容/电感特性
├── 欧姆定律/基尔霍夫定律
├── 运放电路 (放大/滤波/比较)
├── 电源电路 (LDO/DC-DC)
└── 实验: 搭建 LED 驱动电路

Week 2: 数字电路
├── 逻辑门/触发器/计数器
├── 时序逻辑/组合逻辑
├── ADC/DAC 原理
├── 通信接口 (UART/SPI/I2C)
└── 实验: 逻辑分析仪使用
```

### 课程 0.2: C 语言编程
```
Week 3: C 基础
├── 数据类型/运算符/控制流
├── 指针/数组/结构体
├── 函数/递归/宏
├── 内存管理 (栈/堆)
└── 实验: 命令行工具编写

Week 4: C 进阶
├── 位运算/寄存器操作
├── 回调函数/函数指针
├── 链表/队列/状态机
├── Makefile / CMake
└── 实验: 简单 RTOS 任务调度
```

## Level 1: 核心技能 (8 周)

### 课程 1.1: STM32 嵌入式开发 (3 周)
```
Week 5: STM32 基础
├── STM32 架构 (Cortex-M4/M7)
├── 时钟树/电源管理
├── GPIO/中断/定时器
├── UART/SPI/I2C 驱动
├── DMA 传输
└── 实验: LED 流水灯 + 按键中断

Week 6: 中级外设
├── ADC/DAC 采集输出
├── PWM 电机控制
├── 看门狗/低功耗模式
├── Flash 读写/EEPROM 模拟
├── RTOS 任务/队列/信号量
└── 实验: 温度采集 + PID 温控

Week 7: 综合项目
├── Bootloader 设计
├── OTA 升级实现
├── 通信协议设计
├── 状态机框架
└── 项目: 固件升级系统
```

### 课程 1.2: Linux 驱动开发 (3 周)
```
Week 8: Linux 基础
├── Linux 内核架构
├── 字符设备驱动
├── Platform 驱动
├── 设备树 (Device Tree)
└── 实验: 编写 Hello World 驱动

Week 9: 高级驱动
├── I2C/SPI 设备驱动
├── 中断处理 (Top/Bottom Half)
├── DMA 驱动
├── V4L2 摄像头驱动
└── 实验: I2C 传感器驱动

Week 10: 系统集成
├── Buildroot / Yocto 构建
├── 根文件系统制作
├── 系统启动流程
├── 调试工具 (gdb/strace/ftrace)
└── 项目: 完整 Linux 系统构建
```

### 课程 1.3: 硬件设计 (2 周)
```
Week 11: 原理图设计
├── 电源系统设计 (DC-DC/LDO)
├── MCU 最小系统设计
├── 传感器接口设计
├── ESD/EMC 设计
└── 实验: 设计飞控板原理图

Week 12: PCB 设计
├── 叠层设计/阻抗控制
├── 布局/布线规则
├── 电源/地平面设计
├── DFM 检查
└── 实验: 飞控板 PCB Layout
```

## Level 2A: 无人机/手持云台 (6 周)

### 课程 2A.1: 飞控系统 (3 周)
```
Week 13: 传感器与姿态
├── IMU 原理与校准
├── 姿态融合 (互补/Mahony/EKF)
├── 指南针校准与干扰补偿
├── GPS/RTK 定位
└── 实验: 姿态解算 + GPS 定位

Week 14: 控制算法
├── PID 控制器设计与调参
├── 级联控制 (角度→角速度)
├── 电机混控 (四旋翼/六旋翼)
├── 高度控制 (气压计+超声波)
└── 实验: 仿真器 PID 调参

Week 15: 飞控集成
├── 失联保护设计
├── 返航算法
├── 航点导航
├── 数据日志系统
└── 项目: 完整飞控系统集成
```

### 课程 2A.2: 云台与图传 (3 周)
```
Week 16: 云台控制

├── 编码器与传感器融合
├── 跟随/锁定模式
└── 实验: 单轴云台控制

Week 17: 相机系统
├── 图像传感器选型
├── MIPI CSI 接口
├── ISP 管线与 3A 算法
├── V4L2 驱动开发
└── 实验: 相机驱动 + 图像采集

Week 18: 图传系统
├── H.264/H.265 编码
├── 低延迟传输协议
├── FEC/ARQ 纠错
├── 链路预算与天线设计
└── 项目: 低延迟图传链路
```

## 考核体系

### 考核方式
| 级别 | 理论考核 | 实验考核 | 项目考核 | 通过标准 |
|------|----------|----------|----------|----------|
| L0 | 笔试 60% | 实验报告 | - | 60 分 |
| L1 | 笔试 40% | 实验 30% | 项目 30% | 70 分 |
| L2 | 笔试 30% | 实验 30% | 项目 40% | 75 分 |
| L3 | - | - | 产品项目 100% | 评审通过 |

### 项目考核标准
```
项目评审维度:
├── 技术实现 (40%)
│   ├── 功能完整性
│   ├── 代码质量
│   ├── 系统稳定性
│   └── 性能指标
│
├── 产品思维 (30%)
│   ├── 需求分析
│   ├── 用户体验
│   ├── 成本控制
│   └── 可制造性
│
├── 文档规范 (20%)
│   ├── 设计文档
│   ├── 测试报告
│   ├── 用户手册
│   └── 代码注释
│
└── 团队协作 (10%)
    ├── 进度管理
    ├── 沟通协作
    └── 问题解决
```
