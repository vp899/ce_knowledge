### 软件分层架构

```
Layer 5: Application    (用户交互、业务逻辑)
Layer 4: Service        (后台服务、数据管理)
Layer 3: Middleware     (协议栈、引擎、框架)
Layer 2: OS Abstraction (POSIX、CMSIS、API)
Layer 1: OS Kernel      (Linux / FreeRTOS / Zephyr)
Layer 0: Hardware       (SoC、外设、传感器)
```

### 关键设计模式

| 模式 | 适用场景 | 实现 |
|------|----------|------|
| 状态机 | 设备状态管理 | 事件驱动、分层状态机 |
| 观察者 | 事件通知 | 回调函数、发布订阅 |
| 生产者-消费者 | 数据缓冲 | Ring Buffer、消息队列 |
| 中介者 | 模块解耦 | 消息中心、事件总线 |
| 策略 | 算法切换 | 函数指针、接口抽象 |
| 命令 | 操作队列 | 命令模式、任务队列 |
