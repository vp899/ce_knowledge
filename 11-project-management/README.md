## 产品开发流程

```
概念    设计    开发    验证    试产    量产
 │      │      │      │      │      │
 ▼      ▼      ▼      ▼      ▼      ▼
PoC   EVT    DVT    PVT    PP    MP
 │      │      │      │      │      │
概念   工程    设计    生产    小批量  大规模
```

| 阶段 | 英文 | 交付物 | 关键活动 |

| 量产 | MP | 量产产品 | 持续改进、成本优化 |

## 敏捷实践

### Sprint 规划
- Sprint 周期: 2 周
- 计划会议: Sprint 第 1 天，2-4 小时
- 站会: 每天 15 分钟
- 评审: Sprint 最后 1 天
- 回顾: 评审之后 1 小时

### 看板列定义
```
Backlog → To Do → In Progress → Code Review → Testing → Done
```

### DoD (Definition of Done)
- [ ] 代码提交并通过 CI
- [ ] 单元测试覆盖率 >80%
- [ ] 代码评审通过
- [ ] 文档更新
- [ ] 功能测试通过
- [ ] 无 P0/P1 Bug

## 质量管控

### Bug 分级
| 级别 | 定义 | 响应时间 | 修复时间 |
|------|------|----------|----------|
| P0 | 系统崩溃/数据丢失 | 1h | 24h |
| P1 | 核心功能不可用 | 4h | 3 天 |
| P2 | 功能异常/体验差 | 1 天 | 1 周 |
| P3 | 次要问题/建议 | 1 周 | 下个版本 |

### 评审检查点
| 检查点 | 时机 | 内容 |
|--------|------|------|
| 需求评审 | 需求完成后 | 完整性、一致性、可测试性 |
| 设计评审 | 设计完成后 | 架构合理性、接口定义 |
| 代码评审 | 提交时 | 代码质量、安全、性能 |
| 测试评审 | 测试完成后 | 覆盖率、Bug 收敛 |
| 发布评审 | 发布前 | 质量标准、风险评估 |
